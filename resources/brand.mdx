### Logo usage

<Warning>
  Never crop, distort, recolor, rotate, or add effects to the logo. Do not change the relative proportions of the symbol and logotype.
</Warning>

| Do | Don't |
|---|---|
| Use the gradient symbol on light or dark backgrounds | Place the color logo on busy or patterned backgrounds |
| Use the white logo on brand blue surfaces | Recolor the symbol to a non-brand color |
| Use black or white versions for single-color print contexts | Stretch, skew, or rotate the logo |
| Keep clear space around the logo | Use the symbol alone where the full lockup fits |

## Name and usage

The brand name is **vaults.fyi** — always lowercase, always with the dot. Lowercase is intentional, not a typo.

| Context | Form |
|---|---|
| Prose, social, UI copy | `vaults.fyi` |
| Document titles, app store listings, co-marketing tables | `Vaults.fyi` |
| URLs | `vaults.fyi`, `app.vaults.fyi` |

<Warning>
  The dot is part of the name. Never write `vaultsfyi`, `VaultsFYI`, `vaults.FYI`, or `VAULTS.FYI`.
</Warning>

**Sentence starts.** Restructure to avoid opening with the name — `"vaults.fyi launched..."` reads as a typo to most people. Lead with context instead: `"Today we launched..."` or `"The new vaults.fyi feature..."`.

**Products.** `app.vaults.fyi` and the vaults.fyi API are distinct products under the vaults.fyi brand. Use the full product name when referring to a specific one, not just "vaults.fyi."
