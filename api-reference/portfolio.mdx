---
title: "Portfolio"
description: "User portfolio endpoints: positions, idle assets, best deposit options, transaction history, and returns. PRO tier required."
---

<Info>Portfolio endpoints require a **PRO API key**.</Info>

Portfolio endpoints take a user wallet address and return their current vault positions, token balances, transaction history, and yield recommendations. These endpoints power wallet-connected interfaces and portfolio dashboards.

All portfolio endpoints support the standard vault filter parameters (`allowedAssets`, `allowedNetworks`, `minTvl`, etc.) to scope results consistently with your application's vault universe.

---

## GET /v2/portfolio/positions/{userAddress}

Returns all active vault positions for a wallet address.

```bash
GET https://api.vaults.fyi/v2/portfolio/positions/{userAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>
  Wallet address to query positions for.
</ParamField>

### Query parameters

**Vault filters** — same as `/v2/detailed-vaults`:

<ParamField query="allowedAssets" type="array">Assets to include by symbol.</ParamField>
<ParamField query="disallowedAssets" type="array">Assets to exclude by symbol.</ParamField>
<ParamField query="allowedNetworks" type="array" default='["base","mainnet","arbitrum","optimism"]'>Networks to include.</ParamField>
<ParamField query="disallowedNetworks" type="array">Networks to exclude.</ParamField>
<ParamField query="allowedProtocols" type="array">Protocols to include.</ParamField>
<ParamField query="disallowedProtocols" type="array">Protocols to exclude.</ParamField>
<ParamField query="minTvl" type="integer" default="100000">Minimum vault TVL in USD.</ParamField>
<ParamField query="minVaultScore" type="number">Minimum vaults.fyi reputation score.</ParamField>
<ParamField query="onlyTransactional" type="boolean">Only return transactional vaults.</ParamField>
<ParamField query="tags" type="array">Filter by vault tags.</ParamField>
<ParamField query="curators" type="array">Filter by curator name.</ParamField>

**Position filters**:

<ParamField query="apyInterval" type="string" default="7day">APY interval for displayed yield: `1day`, `7day`, or `30day`.</ParamField>
<ParamField query="minUsdAssetValueThreshold" type="number" default="1">Minimum USD position value to include.</ParamField>
<ParamField query="sortBy" type="string">Sort by: `balanceUsd`, `tvl`, `apy1Day`, `apy7Day`, `apy30Day`.</ParamField>
<ParamField query="sortOrder" type="string" default="asc">`asc` or `desc`.</ParamField>

### Response

<ResponseField name="data" type="array">
  <Expandable title="Position object">
    <ResponseField name="address" type="string">Vault contract address.</ResponseField>
    <ResponseField name="name" type="string">Vault name.</ResponseField>
    <ResponseField name="network" type="object">Network: name, chainId, networkCaip.</ResponseField>
    <ResponseField name="asset" type="object">
      <Expandable title="Asset with balance">
        <ResponseField name="address" type="string">Asset address.</ResponseField>
        <ResponseField name="symbol" type="string">Asset symbol.</ResponseField>
        <ResponseField name="decimals" type="integer">Token decimals.</ResponseField>
        <ResponseField name="assetPriceInUsd" type="string">Current asset price.</ResponseField>
        <ResponseField name="balanceNative" type="string">Position balance in native token precision.</ResponseField>
        <ResponseField name="balanceUsd" type="string">Position balance in USD. Omitted if no price available.</ResponseField>
        <ResponseField name="unclaimedNative" type="string">Unclaimed reward balance in native units.</ResponseField>
        <ResponseField name="unclaimedUsd" type="string">Unclaimed reward balance in USD.</ResponseField>
        <ResponseField name="positionValueInAsset" type="string">Position value expressed in the vault's underlying asset.</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="protocol" type="object">Protocol details (name, version, logo, URL).</ResponseField>
    <ResponseField name="apy" type="object">Vault APY breakdown (same shape as detailed-vaults).</ResponseField>
    <ResponseField name="tvl" type="object">Vault TVL (usd, native).</ResponseField>
    <ResponseField name="isTransactional" type="boolean">Supports API-based transactions.</ResponseField>
    <ResponseField name="userEventsIndexed" type="boolean">Whether transaction history is available for this vault.</ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.vaults.fyi/v2/portfolio/positions/0xYourWalletAddress\
?allowedNetworks=mainnet,base&apyInterval=7day"
```

## GET /v2/portfolio/positions/{userAddress}/{network}/{vaultAddress}

Returns the detailed position for a single vault.

```bash
GET https://api.vaults.fyi/v2/portfolio/positions/{userAddress}/{network}/{vaultAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>Wallet address.</ParamField>
<ParamField path="network" type="string" required>Network name or CAIP-2.</ParamField>
<ParamField path="vaultAddress" type="string" required>Vault contract address.</ParamField>

### Query parameters

<ParamField query="apyInterval" type="string" default="7day">APY interval: `1day`, `7day`, or `30day`.</ParamField>

Response is the same shape as a single item from the positions list above.

## GET /v2/portfolio/idle-assets/{userAddress}

Returns all token balances held directly in a wallet (not deposited into vaults). Use this to identify yield opportunities for uninvested assets.

```bash
GET https://api.vaults.fyi/v2/portfolio/idle-assets/{userAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>Wallet address.</ParamField>

### Query parameters

<ParamField query="allowedAssets" type="array">Assets to include by symbol.</ParamField>
<ParamField query="disallowedAssets" type="array">Assets to exclude by symbol.</ParamField>
<ParamField query="allowedNetworks" type="array" default='["base","mainnet","arbitrum","optimism"]'>Networks to scan.</ParamField>
<ParamField query="disallowedNetworks" type="array">Networks to exclude.</ParamField>
<ParamField query="minUsdAssetValueThreshold" type="number" default="0.5">Minimum USD value to include.</ParamField>
<ParamField query="sortBy" type="string">`balanceUsd`.</ParamField>
<ParamField query="sortDirection" type="string" default="asc">`asc` or `desc`.</ParamField>

### Response

```json
{
  "data": [
    {
      "address": "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",
      "symbol": "USDC",
      "name": "USD Coin",
      "decimals": 6,
      "assetPriceInUsd": "1.00",
      "balanceNative": "5000000000",
      "balanceUsd": "5000.00",
      "network": { "name": "mainnet", "chainId": 1, "networkCaip": "eip155:1" }
    }
  ],
  "errors": {
    "unsupportedNetworks": [],
    "unsupportedAssets": []
  }
}
```

## GET /v2/portfolio/best-vault/{userAddress}

Returns the single highest-yielding vault opportunity for a user's current asset balances, subject to your filter criteria.

```bash
GET https://api.vaults.fyi/v2/portfolio/best-vault/{userAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>Wallet address.</ParamField>

### Query parameters

Same vault filter parameters as `/v2/detailed-vaults`, plus:

<ParamField query="apyInterval" type="string" default="7day">APY interval to rank by: `1day`, `7day`, or `30day`.</ParamField>
<ParamField query="minApy" type="number">Minimum APY threshold (raw decimal).</ParamField>
<ParamField query="minUsdAssetValueThreshold" type="number" default="1">Minimum USD wallet balance to consider.</ParamField>

### Response

<ResponseField name="requestedAddress" type="string">The wallet address queried.</ResponseField>
<ResponseField name="asset" type="object">The asset with the best vault opportunity, including current wallet balance.</ResponseField>
<ResponseField name="vault" type="object">The recommended vault, with name, address, network, protocol, TVL, and APY.</ResponseField>

## GET /v2/portfolio/best-deposit-options/{userAddress}

Returns ranked vault recommendations for each idle asset in a wallet — useful for a "deposit suggestions" view.

```bash
GET https://api.vaults.fyi/v2/portfolio/best-deposit-options/{userAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>Wallet address.</ParamField>

### Query parameters

Same vault filter parameters, plus:

<ParamField query="maxVaultsPerAsset" type="integer" default="3">Maximum vault recommendations returned per asset.</ParamField>
<ParamField query="alwaysReturnAssets" type="array">Assets to always include in the response, even if the wallet has no balance.</ParamField>
<ParamField query="minUsdAssetValueThreshold" type="number" default="1">Minimum USD wallet balance to return suggestions for.</ParamField>

### Response

<ResponseField name="requestedAddress" type="string">The wallet address queried.</ResponseField>
<ResponseField name="userBalances" type="array">
  <Expandable title="Per-asset deposit options">
    <ResponseField name="asset" type="object">Asset details including current wallet balance (balanceNative, balanceUsd).</ResponseField>
    <ResponseField name="depositOptions" type="array">
      Ranked vault recommendations for this asset. Each entry includes the full vault object (name, address, network, protocol, APY, TVL, transactional status).
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.vaults.fyi/v2/portfolio/best-deposit-options/0xYourWalletAddress\
?allowedNetworks=mainnet,base&onlyTransactional=true&maxVaultsPerAsset=3"
```

## GET /v2/portfolio/total-returns/{userAddress}/{network}/{vaultAddress}

Returns the total yield earned by a user in a specific vault (net of deposits and withdrawals).

```bash
GET https://api.vaults.fyi/v2/portfolio/total-returns/{userAddress}/{network}/{vaultAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>Wallet address.</ParamField>
<ParamField path="network" type="string" required>Network name or CAIP-2.</ParamField>
<ParamField path="vaultAddress" type="string" required>Vault contract address.</ParamField>

### Response

```json
{
  "address": "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48",
  "symbol": "USDC",
  "name": "USD Coin",
  "decimals": 6,
  "assetPriceInUsd": "1.00",
  "returnsNative": "183.420000"
}
```

<ResponseField name="returnsNative" type="string">
  Total yield earned, in native asset units. This is the difference between current position value and net capital deposited.
</ResponseField>

## GET /v2/portfolio/events/{userAddress}/{network}/{vaultAddress}

Returns the full deposit and withdrawal history for a user's position in a vault.

```bash
GET https://api.vaults.fyi/v2/portfolio/events/{userAddress}/{network}/{vaultAddress}
```

### Path parameters

<ParamField path="userAddress" type="string" required>Wallet address.</ParamField>
<ParamField path="network" type="string" required>Network name or CAIP-2.</ParamField>
<ParamField path="vaultAddress" type="string" required>Vault contract address.</ParamField>

### Response

<ResponseField name="asset" type="object">Asset details (address, symbol, decimals, price).</ResponseField>
<ResponseField name="lpToken" type="object">LP token details (address, symbol, decimals).</ResponseField>
<ResponseField name="items" type="number">Total number of events.</ResponseField>
<ResponseField name="data" type="array">
  <Expandable title="Event">
    <ResponseField name="timestamp" type="integer">Event Unix timestamp (seconds).</ResponseField>
    <ResponseField name="blockNumber" type="string">Block number.</ResponseField>
    <ResponseField name="transactionHash" type="string">Transaction hash.</ResponseField>
    <ResponseField name="logIndex" type="number">Log index within the transaction.</ResponseField>
    <ResponseField name="eventType" type="string">`deposit` or `withdrawal`.</ResponseField>
    <ResponseField name="assetAmountNative" type="string">Asset amount in native units.</ResponseField>
    <ResponseField name="lpTokenAmount" type="string">LP tokens minted (deposit) or burned (withdrawal).</ResponseField>
    <ResponseField name="sharePrice" type="number">Vault share price at the time of the event.</ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.vaults.fyi/v2/portfolio/events/0xYourWalletAddress/mainnet/0xVaultAddress"
```

      "transactionHash": "0xabc...",
      "logIndex": 12,
      "eventType": "deposit",
      "assetAmountNative": "10000000000",
      "lpTokenAmount": "9179320000000000000000",
      "sharePrice": 1.0891
    }
  ]
}
