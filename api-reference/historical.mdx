---
title: "Historical"
description: "Time-series APY, TVL, share price, and asset price data for any vault. PRO tier required."
---

<Info>Historical endpoints require a **PRO API key**. Basic keys will receive a 403 response.</Info>

All historical endpoints are paginated and support timestamp filtering. Timestamps are **Unix seconds**. APY values are raw decimals — multiply by 100 to display as a percentage.

---

## GET /v2/historical/{network}/{vaultAddress}

Returns combined APY, TVL, and share price time-series for a vault in a single call.

```bash
GET https://api.vaults.fyi/v2/historical/{network}/{vaultAddress}
```

### Path parameters

<ParamField path="network" type="string" required>
  Network name (`mainnet`, `arbitrum`, `base`, etc.) or CAIP-2 identifier.
</ParamField>
<ParamField path="vaultAddress" type="string" required>
  Vault contract address.
</ParamField>

### Query parameters

<ParamField query="page" type="integer" default="0">Page number (zero-indexed).</ParamField>
<ParamField query="perPage" type="integer" default="50">Items per page.</ParamField>
<ParamField query="apyInterval" type="string" default="7day">
  APY smoothing window: `1day`, `7day`, or `30day`.
</ParamField>
<ParamField query="granularity" type="string" default="1hour">
  Data point frequency: `1hour`, `1day`, or `1week`.
</ParamField>
<ParamField query="fromTimestamp" type="integer" default="0">
  Start of the time range (Unix seconds).
</ParamField>
<ParamField query="toTimestamp" type="integer">
  End of the time range (Unix seconds). Defaults to now.
</ParamField>

### Response

<ResponseField name="itemsOnPage" type="integer">Items on this page.</ResponseField>
<ResponseField name="nextPage" type="integer">Next page number, or null.</ResponseField>
<ResponseField name="data" type="array">
  <Expandable title="Data point">
    <ResponseField name="timestamp" type="integer">Unix timestamp (seconds) of this data point.</ResponseField>
    <ResponseField name="blockNumber" type="string">Block number at this data point.</ResponseField>
    <ResponseField name="apy" type="object">
      <Expandable title="APY breakdown">
        <ResponseField name="base" type="number">Base APY (raw decimal).</ResponseField>
        <ResponseField name="reward" type="number">Reward APY (raw decimal).</ResponseField>
        <ResponseField name="total" type="number">Total APY (raw decimal).</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="tvl" type="object">
      <Expandable title="TVL breakdown">
        <ResponseField name="usd" type="string">TVL in USD.</ResponseField>
        <ResponseField name="native" type="string">TVL in native asset units.</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="sharePrice" type="number">Vault share price at this point.</ResponseField>
  </Expandable>
</ResponseField>

### Example — 90-day daily APY history

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.vaults.fyi/v2/historical/mainnet/0x9D39A5DE30e57443BfF2A8307A4256c8797A3497/apy\
?apyInterval=7day&granularity=1day&perPage=90\
&fromTimestamp=1735689600&toTimestamp=1743465600"
```

## GET /v2/historical/asset-prices/{network}/{assetAddress}

Returns historical USD prices for any tracked asset on a given network.

```bash
GET https://api.vaults.fyi/v2/historical/asset-prices/{network}/{assetAddress}
```

### Path parameters

<ParamField path="network" type="string" required>Network name or CAIP-2.</ParamField>
<ParamField path="assetAddress" type="string" required>Asset contract address.</ParamField>

### Query parameters

<ParamField query="page" type="integer" default="0">Page number.</ParamField>
<ParamField query="perPage" type="integer" default="50">Items per page.</ParamField>
<ParamField query="granularity" type="string" default="1hour">`1hour`, `1day`, or `1week`.</ParamField>
<ParamField query="fromTimestamp" type="integer">Start timestamp (Unix seconds).</ParamField>
<ParamField query="toTimestamp" type="integer">End timestamp (Unix seconds).</ParamField>

### Response

```json
{
  "itemsOnPage": 30,
  "nextPage": null,
  "data": [
    {
      "timestamp": 1735689600,
      "priceInUsd": "3241.58"
    }
  ]
}
```

## Pagination pattern

All historical endpoints use cursor-style pagination. Fetch subsequent pages by incrementing `page` until `nextPage` is null:

```js
async function fetchAll(url, apiKey) {
  let page = 0, results = [];
  while (true) {
    const r = await fetch(`${url}&page=${page}`, { headers: { 'x-api-key': apiKey } });
    const { data, nextPage } = await r.json();
    results.push(...data);
    if (nextPage == null) break;
    page = nextPage;
  }
  return results;
}
```
